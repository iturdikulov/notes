How to Learn Nix, Part 19:
Switching from Homebrew to Nix

I don‚Äôt feel like I know enough about Nix to manage my own build dependencies with it, but I do feel like I know enough to use Nix as a simple package manager. I use Homebrew on my laptop, and at this point I definitely know more about Nix than I do about Homebrew.1

Why would I switch away from Homebrew? Mostly because I think it will be educational. I don‚Äôt really have anything against Homebrew. It works. It has a lot of the packages I want. It does break more often than I‚Äôd like ‚Äì not, like, often, but any time I go more than a year or so without using a particular computer I know that I‚Äôm going to return to some bizarre Ruby error that I have to spend half an hour figuring out before I can do anything.

And there‚Äôs no way that Nix would ever play me like that.

Homebrew doesn‚Äôt seem to distinguish between ‚Äúpackages I want to have installed‚Äù and ‚Äúpackages I have to have installed because they are dependencies of other packages.‚Äù

I remember this bothering me before. It‚Äôs bothering me a lot right now, as I try to get rid of it.

So it‚Äôs worth asking what I care about here. Why do I use brew info? I ponder this for a moment, and come up with a few reasons:

  • to check if a specific package name is the actual thing I want before I install it (i.e. cabal vs cabal-install)
  • to read the package‚Äôs description (usually because I encountered some unfamiliar command in a blog post somewhere: this is a very fast way to find out what it is)
  • to see what version of the package Homebrew has (especially in the case of compilers and development tools, where I care deeply about the exact version I‚Äôm using)

Those are the only reasons I can think of. So honestly the first two lines of brew info‚Äôs output cover 100% of what I want.

I don‚Äôt think I ever really realized that. Turning back to Nix, how can I get those two lines?

Alright. Let‚Äôs move on: uninstall and install. I know this:

$ nix-env -iA nixpkgs.git
$ nix-env -e git

Except that the string ‚Äúgit‚Äù means a different thing when you use -iA than it does when you use -e.4 You install packages by their ‚ÄúNixpkgs attribute name‚Äù or whatever, but you uninstall them by their ‚Äúname.‚Äù You can install by ‚Äúname‚Äù but it takes 30 seconds so you will never do that. There is no way to uninstall by ‚Äúattribute.‚Äù

I understand why this is, now: the attribute path in Nixpkgs is not an intrinsic part of the package. The package doesn‚Äôt know that it‚Äôs called nixpkgs.git. You could evaluate nixpkgs.git and find its store path, but that wouldn‚Äôt work in the case that nixpkgs.git has changed since you installed it (because of nix-channel --update, say, or because you renamed the channel, or whatever).

So like‚Ä¶ this is a weird gross problem that Homebrew doesn‚Äôt have.

Or does it? Have you ever tried to uninstall a package that has been removed from Homebrew? Or renamed? Or to uninstall a package that was installed with an older version of Homebrew and now you‚Äôre getting weird Ruby errors because the format of casks has changed in a backwards-incompatible way?

You haven‚Äôt? Oh. Yeah; to be fair it is pretty rare.
